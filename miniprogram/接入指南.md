# 记账本微信小程序 — 接入指南

与现有 H5（`book/`）共用同一个 Supabase 项目和 `ledgers` 表，**家人数据互通**。
个人主体即可，免费，自用/家人用走「开发版 + 体验版」，无需上架审核。

---

## 一、注册小程序（拿 AppID）

1. 打开微信公众平台 https://mp.weixin.qq.com → 右上角「立即注册」→ 选 **小程序**
2. 用一个**没注册过公众号/小程序的邮箱**注册并激活
3. 主体类型选 **个人**（免费、当天可用）
4. 进入后台：**开发管理 → 开发设置**，复制 **AppID（小程序ID）**

> 个人主体不能开通微信支付、部分类目受限，但记账本（工具/效率类）自用完全不受影响。

## 二、安装微信开发者工具

下载页：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

- 选「稳定版 Stable → macOS」，按芯片选 ARM64（Apple 芯片）或 x64（Intel）
- 打开 `.dmg` 把图标拖进「应用程序」
- 首次打开若提示无法验证开发者： → 系统设置 → 隐私与安全性 → 「仍要打开」
- 用注册小程序的微信扫码登录

## 三、导入项目

1. 开发者工具 → 「导入项目」
2. 目录选本仓库的 `miniprogram/` 文件夹
3. AppID 填第一步复制的值
4. 「详情 → 本地设置」勾选 **不校验合法域名**（开发阶段免配域名直接跑）

## 四、配置合法域名（真机/家人用之前必做）

小程序后台 **开发管理 → 开发设置 → 服务器域名 → request 合法域名**，添加：

```
https://kjasiqqtihagwsnthbvc.supabase.co
```

（必须 HTTPS；这正是现有 Supabase 域名，所以小程序与 H5 同一份数据）

## 五、发给家人

1. 开发者工具点 **预览** → 手机微信扫码，自己先真机测试
2. 点 **上传** → 后台 **版本管理** 把该版本设为「体验版」
3. 后台 **成员管理 → 体验成员** 加入家人微信
4. 家人打开体验版；或在小程序里点「分享 → 转发给家人」，对方点开自动加入同一账本

## 配置文件

- `config.js`：Supabase `supabaseUrl` / `supabaseAnonKey`，已填好（与 H5 同源）
- `config.example.js`：模板，换 Supabase 项目时参考

## 数据与同步说明

- 数据结构与 H5 完全一致：`{ budgetByMonth, months: { 'YYYY-MM': { income, savings, expenses } } }`
- 同步：打开/切前台时拉取云端，前台每约 15s 轮询一次；本地改动即时上云
- 账本编号 `bk_xxx` 是共享钥匙，和 H5 通用：H5 里加入同一编号即可看到小程序记的数据，反之亦然

## 功能对照（与 H5 一致）

- 收入 / 储蓄 / 开支三类明细的增删改
- 开支：计划金额、记支出（多次部分支出）、勾选完成 / 再点取消完成
- 开支列表排序：已完成 → 部分支出 → 待支出；已完成按勾选时间从新到旧
- 预算总额随收入自动同步，可手动覆盖
- 汇总：预算总额 / 预算支出 / 已支出 / 还剩 + 进度条
