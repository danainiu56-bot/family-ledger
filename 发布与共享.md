# 记账本 — 发布与多人共享

## 一、发布到 GitHub Pages

1. 确认 `scripts/pages/bookkeeping-config.js` 已配置（见下文第二节）
2. 提交并 push 到 `main` 分支
3. 在 GitHub 仓库 **Settings → Pages** 中，Source 选 **GitHub Actions**
4. push 后 Actions 会自动部署，访问地址：

**微信推荐（无中文路径、无 .html 后缀）：**

```
https://danainiu56-bot.github.io/family-ledger/book/
```

**国内 CDN 备用：**

```
https://cdn.jsdelivr.net/gh/danainiu56-bot/family-ledger@bookkeeping/
```

若已启用 GitHub Actions 部署，根路径也会打开记账本：

```
https://danainiu56-bot.github.io/family-ledger/
```

旧地址 `记账本.html` 会自动跳转到 `book/`。

5. 把链接发到微信「文件传输助手」，手机点开即可使用

> 微信会拦截 TinyURL、bit.ly 等第三方短链，请直接发上述链接。若提示「非微信官方网页」，点「继续访问」即可。

## 二、多人共享同一账本（Supabase 免费版）

当前应用数据默认在各自手机本地。要**多人看同一份账**，需要接 Supabase 云端存储。

### 1. 创建 Supabase 项目

1. 打开 [https://supabase.com](https://supabase.com) 注册并新建项目（免费）
2. 进入 **Project Settings → API**，复制：
   - **Project URL** → `supabaseUrl`
   - **anon public key** → `supabaseAnonKey`

### 2. 建表

在 Supabase **SQL Editor** 执行：

```sql
create table if not exists ledgers (
  id text primary key,
  data jsonb not null default '{}',
  updated_at timestamptz not null default now()
);

alter table ledgers enable row level security;

create policy "allow anon access"
  on ledgers for all
  using (true)
  with check (true);

-- 启用实时同步（家人改账后立刻刷新）
alter publication supabase_realtime add table public.ledgers;
```

### 3. 写入配置

```bash
cp scripts/pages/bookkeeping-config.example.js scripts/pages/bookkeeping-config.js
```

编辑 `scripts/pages/bookkeeping-config.js`，填入 URL 和 anon key，然后 commit 并 push。

### 4. 使用方式

1. 你：打开网页 → 点「创建账本」→ 得到编号（如 `bk_a1b2c3d4`）
2. 点「分享链接」，发给家人/同事
3. 对方：打开链接自动加入同一账本，数据实时同步（约 15 秒刷新）

> 账本编号相当于钥匙，请勿公开到互联网。适合家庭/小团队内部使用。
